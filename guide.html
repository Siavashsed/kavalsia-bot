Kavalsia Content Network

A self-hosted AI content engine that writes, publishes, and manages a network of niche blog sites — fully controlled from your browser or Telegram.

AI-Powered GitHub Pages Telegram Control

The system takes a list of sites you configure, generates full articles using Claude AI, fetches matching images from Pexels, builds HTML pages, and pushes them directly to GitHub Pages — no CMS, no WordPress, no hosting bills.

Quick start: Go to Setup → fill in your API keys → go to Hub → click Run Now. That's it.

File Structure

Step-by-Step Setup

  1. Prerequisites

    You need the following before anything will work:

    • Python 3.10+ — check with python3 --version
    • Anthropic API key — from console.anthropic.com
    • GitHub Personal Access Token — needs repo scope. Create at GitHub → Settings → Developer Settings → Personal access tokens → Tokens (classic)
    • Pexels API key (optional but recommended) — free at pexels.com/api for article images
    • One or more GitHub Pages repositories — create a public repo, go to Settings → Pages → Source: Deploy from branch → main

  2. Install Python dependencies

    Open a terminal in the content-network/ folder and run:

    pip install anthropic requests flask

    If you plan to use Google Indexing API, also run:

    pip install google-auth google-auth-httplib2

  3. Set environment variables

    The engine reads secrets from environment variables — never hardcode them. Set these before starting the server:

    # Required
export ANTHROPIC_API_KEY="sk-ant-..."
export GITHUB_TOKEN="ghp_..."

# Recommended
export PEXELS_API_KEY="your_pexels_key"

# Optional
export BREVO_API_KEY="your_brevo_key"      # newsletter emails
export TAVILY_API_KEY="your_tavily_key"    # web search grounding
    Mac/Linux: Add these to your ~/.zshrc or ~/.bashrc so they persist across sessions. On Windows, use System → Environment Variables.

  4. Configure your sites

    Open launcher.html in your browser (double-click the file, or serve it via the local server — see step 5).

    • Enter a password for the admin pages
    • Add each site: domain, GitHub repo (username/repo-name), category, tone, persona
    • Click Save Configuration

    This writes everything to network-config.json.

  5. Start the local server

    The browser pages talk to a local Flask server. Start it with:

    python server.py

    You should see:

     * Running on http://127.0.0.1:5050

    Now open any admin page in your browser. The server must stay running while you use the admin UI.

    Keep it running: Use a terminal multiplexer like tmux or screen, or a process manager like pm2, so the server stays up when you close the terminal.

  6. Publish your first article

    • Open hub.html
    • Log in with your password
    • Click Run Now
    • Watch the live log — it will generate, upload, and confirm each article
    • Your GitHub Pages site will update within 30–60 seconds of the push
    To publish a specific topic, enter it in the Topic Override field before clicking Run Now.

Run a job

  • Site filter — run only specific sites (by domain or ID)

Editor

The Editor is where you manage the AI brain of the network — prompts, site configs, global scripts, affiliate links, and social media.

Global settings (top section)

Field What it does
Global AI Prompt Added to every article on every site. Use for brand voice, SEO rules, linking rules.
Global Negative Prompt Injected at the very top of every prompt as ABSOLUTE PROHIBITIONS. Pre-filled with em-dash ban + AI-cliché list.
Global Header Scripts Injected into <head> on every page of every site. Use for GA4, pixels, consent banners.
Global Footer Scripts Injected before </body> on every page of every site.
Affiliate Links Network-wide keyword → URL map. Claude-generated text matching these keywords gets wrapped in affiliate links.

Per-site settings

Click any site card to expand it. Each site has its own:

  • Domain, GitHub repo, category, tone, persona
  • System prompt override — site-specific instructions added after the global prompt
  • Newsletter — Brevo list ID and sender details
  • Site-level affiliate links — stacks on top of global affiliate links
  • Social media — webhook, Twitter, LinkedIn toggle and tokens
  • Header/footer scripts — site-level scripts (stacked after global scripts)
  • Audience warming — gradual audience expansion schedule
Script stacking order: Global scripts always load first. Site-level scripts load after. This means GA4 (global) initializes before any site-specific pixel.

Dashboard

A read-only view of all published articles across every site. Use it to:

  • Browse recent articles by site
  • Preview the rendered article HTML in a mobile/desktop frame
  • Verify design, links, and formatting before checking the live site

The preview renderer uses the same themes as bot.py, so what you see here matches what's on the live site.

Global AI Prompt

Everything in the Global AI Prompt is appended to every article generation call. Use it for:

  • Brand voice rules ("write in a confident, direct tone")
  • SEO requirements ("include the target keyword in H1, first paragraph, and one H2")
  • Structural rules ("always include a FAQ section at the end")
  • Internal linking rules ("link to the homepage when mentioning the domain")
# Example global prompt
Write in a confident, direct, expert tone. Never use passive voice.
Always include an FAQ section (minimum 4 questions) at the end of each article.
Structure: H1 → intro paragraph → 3–5 H2 sections → FAQ → conclusion.
Target keyword must appear in H1, first 100 words, and at least one H2.
Word count: 1,200–1,800 words.

Global Negative Prompt

The negative prompt is injected at the very top of every article prompt, before anything else, as an absolute prohibition block. Claude sees it first and cannot override it with later instructions.

Default value (pre-filled in Editor):

DO NOT EVER USE " —" (em-dash). NEVER use: delve into, it's worth noting,
in the realm of, game-changing, harness the power, leverage (as metaphor),
navigate the landscape, cutting-edge, comprehensive, robust, furthermore,
in conclusion, in summary, to summarize, it is important to note,
as an AI language model.
Add any phrase that consistently appears in AI-generated content and reads as artificial. Each line or phrase is treated as a prohibition.

Affiliate Links

The system automatically wraps matching keywords in affiliate links with rel="sponsored noopener".

How to add affiliate links

  1. Open Editor
  2. In the Global section, find Affiliate Links
  3. Click + Add Link
  4. Enter the keyword (e.g. VPN service) and the destination URL
  5. Click Save Config

How it works

  • Longest keyword is matched first (so "best VPN service" matches before "VPN service")
  • Only first occurrence per keyword is linked (avoids over-linking)
  • Links are added after HTML generation, so they never appear inside headings or existing links
  • Site-level affiliate links are merged with global ones — site wins on conflict
Keep keyword phrases specific. VPN service is better than just VPN — less chance of accidentally linking something you didn't intend.

GitHub Actions (Scheduled Publishing)

Run the content engine on a schedule without keeping your computer on.

Create the workflow file

In your content-network repository (not the sites — the repo containing bot.py), create:

.github/workflows/publish.yml
name: Publish Content

on:
  schedule:
    - cron: '0 8 * * 1,3,5'   # Mon/Wed/Fri at 8am UTC
  workflow_dispatch:            # also allows manual runs from GitHub UI

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: pip install anthropic requests flask

      - name: Run content bot
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GITHUB_TOKEN:      ${{ secrets.GH_PAT }}
          PEXELS_API_KEY:    ${{ secrets.PEXELS_API_KEY }}
          BREVO_API_KEY:     ${{ secrets.BREVO_API_KEY }}
        run: python bot.py

Add secrets to GitHub

  1. Go to your content-network repo on GitHub
  2. Settings → Secrets and variables → Actions
  3. Add: ANTHROPIC_API_KEY, GH_PAT (your GitHub token), PEXELS_API_KEY, BREVO_API_KEY
Use GH_PAT, not GITHUB_TOKEN for your personal access token. GitHub Actions auto-injects its own GITHUB_TOKEN — if you use that name your secret will be ignored and pushes to other repos will fail.

Weekly ideas via GitHub Actions

To run the Telegram ideas workflow on a schedule:

- name: Weekly ideas
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
    TELEGRAM_BOT_TOKEN: ${{ secrets.TELEGRAM_BOT_TOKEN }}
    TELEGRAM_CHAT_ID:   ${{ secrets.TELEGRAM_CHAT_ID }}
  run: python telegram_bot.py ideas

Web Search Grounding

Before generating each article, the bot can search the web for current information and inject it as context. This produces more accurate, timely content.

Enable

In Editor → Global Settings → Settings, set:

"web_search": {
  "enabled": true,
  "provider": "tavily",
  "tavily_api_key": "tvly-..."
}

Alternatively use Perplexity:

"provider": "perplexity",
"perplexity_api_key": "pplx-..."

Get a Tavily key at tavily.com (free tier: 1,000 searches/month).

Google Indexing API

After publishing, the bot can ping Google to request immediate indexing of new articles.

Setup

  1. Go to Google Cloud Console → create a project
  2. Enable the Indexing API
  3. Create a Service Account → download the JSON key
  4. Save the key as google-service-account.json in your content-network folder
  5. In network-config.json: "google_indexing": {"enabled": true, "service_account_path": "google-service-account.json"}
  6. Add the service account email to your site's Google Search Console as an Owner
Google only accepts indexing pings for sites where the service account is verified in Search Console as an Owner (not just a User).

Content Workflow

Here's what happens every time a publish job runs, step by step:

  1. Load config — reads network-config.json, filters active sites
  2. For each active site:
  3. Select topic — uses topic override if provided, otherwise uses a rotating topic from the site's topic pool or auto-generates one
  4. Web search (if enabled) — queries Tavily/Perplexity for current information on the topic
  5. Find internal links — loads existing articles, scores by keyword overlap with the new topic, passes top 3 as suggestions
  6. Generate article — sends prompt to Claude with: negative prohibitions → global prompt → site prompt → web context → internal link suggestions → topic
  7. Fetch image — queries Pexels with the article's image_query field
  8. Build HTML — generates full article page HTML with theme, image, scripts, affiliate links injected
  9. Push to GitHub — pushes article HTML, updates index.html (homepage), updates articles.json
  10. Send newsletter (if enabled) — sends Brevo campaign to list
  11. Broadcast social (if enabled) — posts to webhook/Twitter/LinkedIn
  12. Ping Google (if enabled) — requests immediate indexing of the new URL

Cloudflare Proxy — Hide Your Host IP (Free)

You do not need different hosting for each site. Put Cloudflare's free proxy in front of every domain instead. All sites will resolve to Cloudflare's shared IP ranges — which millions of sites share — making it impossible for Google to identify your origin server or detect that multiple domains share the same host.

Why this works

When Cloudflare proxy is enabled (orange cloud), all DNS requests return Cloudflare's IP, not your server's IP. Google's crawlers see Cloudflare IPs, which are completely normal. Your actual origin (GitHub Pages, a VPS, anything) is invisible.

Setup — 5 minutes per domain

  1. Go to cloudflare.com → Add a site → enter your domain
  2. Select the Free plan
  3. Cloudflare scans your DNS. You'll see your existing records.
  4. For the main A/CNAME record, make sure the cloud icon is orange (proxied). Click it to toggle if it's grey.
  5. Change your domain's nameservers to the two Cloudflare nameservers shown (e.g. ken.ns.cloudflare.com)
  6. Done. Propagation takes 5–30 minutes.

For GitHub Pages specifically

  1. In your GitHub repo → Settings → Pages → set custom domain to your domain
  2. In Cloudflare DNS, add a CNAME record: www → YOUR_GITHUB_USERNAME.github.io (proxied ☁)
  3. Add an A record for the root domain pointing to GitHub Pages IPs, or redirect root to www — Cloudflare's Page Rules can handle this for free
  4. In Cloudflare SSL/TLS, set to Full (not Flexible)

GitHub Pages IPs: 185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153

20 sites, one Cloudflare account

You can manage all 20+ domains from a single free Cloudflare account. Each domain gets its own zone (free tier allows unlimited domains). The proxy is free on all of them. You can also keep all sites on a single cheap VPS (Hetzner €4/month, DigitalOcean $6/month) — with Cloudflare in front, Google sees nothing unusual.

Additional Cloudflare free benefits

  • Free SSL on every domain — required for AdSense and trust signals
  • DDoS protection — your sites stay up under any traffic
  • Cloudflare Analytics — basic traffic stats without cookies (GDPR-friendly)
  • Caching — pages served from Cloudflare's edge, faster globally
  • Page Rules — redirect root → www, force HTTPS, etc.

Backlink Content Generator

Every time an article is published, the bot automatically generates ready-to-post content for Reddit, LinkedIn, and Quora — saved to backlinks/{slug}.md in each site's repo. These create real backlinks from high-authority platforms (Reddit DA91, LinkedIn DA98, Quora DA92) without cross-linking your own network sites to each other.

What gets generated per article

  • Reddit — 3 relevant subreddits suggested + a title and post body (150-200 words, written as a genuine community member, not self-promotion)
  • LinkedIn — A hook line + post body (120-150 words) + 5 hashtags. Post manually or schedule via Buffer
  • Quora — A question the article answers + a 250-300 word authoritative answer with a natural link back

How to use the backlink files

  1. After each bot run, go to your site repo → backlinks/ folder
  2. Open the {slug}.md file for the article just published
  3. Copy the Reddit section → post to the suggested subreddits (do 1-2 per article, not all 3 at once)
  4. Copy the LinkedIn section → post to your persona's LinkedIn account
  5. Copy the Quora section → search the question on Quora, answer it
  6. Repeat consistently. After 30-50 articles you'll have 30-50 real backlinks from high-DA platforms.

Reddit posting rules — don't get banned

  • Post to a subreddit only 1-2 times per week per account
  • Make sure the account is at least 30 days old and has some comment karma first
  • Post the content genuinely — if it's a good answer, Reddit will upvote it. If it reads as spam, it gets removed.
  • Never post the same content to multiple subreddits on the same day
  • Use a dedicated Reddit account per site persona (Marcus Chen has his own account, etc.)

Monetization Setup

AdSense — ads.txt + publisher ID

  1. Apply for Google AdSense at adsense.google.com — one account covers all your sites

  3. In network-config.json, set "adsense_publisher_id" per site

  2. Get the form URL (e.g. https://app.brevo.com/page/xyz) or embed code
  3. In network-config.json, set "signup_url" per site to that URL

Troubleshooting

"Config not found" or "Could not connect to server"

The browser pages talk to server.py on port 5050. Make sure you started it with python server.py and it's still running. Check the terminal for errors.

Articles push but the site doesn't update

GitHub Pages can take 30–90 seconds to deploy. Check the Actions tab in the GitHub repo to see if Pages is building. If it's stuck, make sure the repo has Pages enabled (Settings → Pages → Source: Deploy from branch → main → / (root)).

GitHub push returns 404 or 403

Your GitHub token doesn't have access to the repo, or has expired. Create a new token with repo scope at GitHub → Settings → Developer settings → Personal access tokens.

Articles are generating but no image appears

Check your PEXELS_API_KEY environment variable. Without it, articles publish with no image. The Pexels free tier allows 200 requests/hour.

Telegram bot doesn't respond

  1. Make sure "enabled": true in settings.telegram
  2. Verify the token with: curl https://api.telegram.org/botYOUR_TOKEN/getMe
  3. Check that your chat ID is in allowed_chat_ids — get it by messaging @userinfobot
  4. Make sure only one instance of telegram_bot.py is running (multiple instances conflict)

GitHub Actions run but nothing is published

Check the Actions log. Common causes: secrets not set, GITHUB_TOKEN name collision (use GH_PAT instead), or network-config.json not committed to the repo.

Important: network-config.json must be committed to the repo so GitHub Actions can read it. But don't commit API keys inside it — use environment variables instead.

Claude returns invalid JSON

Occasionally Claude's response can't be parsed. The bot logs the raw response and skips that article. It usually resolves on the next run. If it happens consistently, simplify the global prompt or reduce max_tokens.
