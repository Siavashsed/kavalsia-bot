📖 Complete Deployment Guide — From Zero to Live Network
Follow these steps exactly in order. Each step depends on the previous one. Estimated total setup time: 2–4 hours for a full 20-site network.
1
Register Your Domains
One domain per site — do this first
Recommended registrars (cheapest to most expensive):
• Porkbun — Best prices, clean UI. Most .com domains ~$9-11/yr. Great for bulk registration.
• Namecheap — Reliable, good dashboard, includes WhoisGuard privacy free.
• GoDaddy — More expensive, but familiar if you're already there.

What to register:
• For each of your 20 sites, register a .com domain that matches the niche (e.g. cryptoinsightsreport.com)
• Avoid hyphens — they look spammy to Google
• Choose short, memorable names with keywords from the site's category

After registering: Copy all 20 domain names into Step 3 of this launcher (Configure Network Sites → Basic tab for each site).
2
Set Up Cloudflare (Free)
DNS + CDN + Pages hosting — all free
Create a free Cloudflare account at cloudflare.com

For EACH domain:
1. Cloudflare dashboard → Add a Site → enter your domain → select Free plan
2. Cloudflare scans your existing DNS records — click Continue
3. Cloudflare gives you 2 nameservers (e.g. blake.ns.cloudflare.com)
4. Go to your registrar (Porkbun/Namecheap) → find the domain → change nameservers to the Cloudflare ones
5. Back in Cloudflare, click Done, check nameservers
6. Wait 5-30 minutes for DNS to propagate (usually fast with Porkbun/Namecheap)

Tip: You can batch this — add all 20 domains to Cloudflare, change nameservers at the registrar for all 20, then come back to check status. Saves a lot of back-and-forth.
3
Create GitHub Repositories
1 bot repo + 1 repo per site = 21 total
Sign in to GitHub (github.com). Create a free account if needed.

Create the BOT repository (the engine):
1. New repository → name it content-network-bot → set to Private → Create
2. Upload these files from your computer: bot.py, telegram_bot.py, requirements.txt, network-config.json
3. Also upload the .github/workflows/ folder (which contains daily-post.yml and weekly-ideas.yml)

Create ONE repository per site (20 repos):
1. Name each repo to match the domain slug (e.g. cryptoinsights-site)
2. Set to Public (required for Cloudflare Pages free tier)
3. Initialize with a README (just click the checkbox — you'll replace it)
4. Enter the exact repo name (username/reponame) in this launcher's Step 3 for each site

Create a Personal Access Token (PAT):
1. GitHub → Settings (top-right) → Developer settings → Personal access tokens → Tokens (classic)
2. Generate new token → name it "Content Network Bot" → Expiration: 1 year
3. Scopes: check repo (full control of repos) — that's all you need
4. Copy the token immediately — paste it into Step 1 of this launcher (GitHub Token field). You won't see it again.
4
Connect Sites to Cloudflare Pages
Free hosting with auto-deploy on every push
For EACH of your 20 site repos:
1. Cloudflare dashboard → Pages (left sidebar) → Create a project
2. Select Connect to Git → authorize GitHub → select the site's repo
3. Build settings: Framework preset = None — leave build command and output directory empty
4. Click Save and Deploy — Cloudflare deploys immediately
5. Your site is now live at your-repo-name.pages.dev

Connect your custom domain:
1. In the Pages project → Custom domains → Set up a custom domain
2. Enter your domain (e.g. cryptoinsights.com)
3. Cloudflare automatically creates the DNS CNAME record (since your domain is already on Cloudflare)
4. Wait 1-2 minutes → your domain now points to Cloudflare Pages

Tip: Every time the bot pushes new HTML to the GitHub repo, Cloudflare Pages auto-deploys within 30 seconds. Zero maintenance required.
5
Get All API Keys
Anthropic + Pexels + Brevo + Telegram
Anthropic (Claude API) — REQUIRED:
• console.anthropic.com → API Keys → Create Key → paste into Step 1 of this launcher
• Add a payment method. Cost: ~$0.50-1.00 per day for 20 sites at 1 article each

Pexels (Stock Photos) — FREE:
• pexels.com/api → Request Access → get API key → paste into Step 1
• 200 requests/hour free — more than enough

Brevo (Newsletter Email) — FREE:
• brevo.com → Sign up free → Settings → API Keys → Generate → paste into Step 1
• Create a separate contact list for each site in Brevo → get the list ID → enter in Step 3 per-site Basic tab
• 300 free emails/day across all lists

Telegram Bot (Weekly Ideas) — FREE:
• Open Telegram → search @BotFather → send /newbot → follow prompts → get your bot token
• Start a chat with your new bot → send any message → visit api.telegram.org/bot{YOUR_TOKEN}/getUpdates to find your chat ID
• Paste both token and chat ID into Step 1 of this launcher

9
Trigger the First Run & Verify
Don't wait until 8am UTC — test it now
Trigger manually from GitHub:
1. Bot repo → Actions tab → select Daily Content Publisher
2. Click Run workflow → Run workflow (green button)
3. Watch the run logs expand in real time — you'll see each site being processed

What success looks like:
• Each site shows "Article pushed: True → /slug/" in the logs
• Your GitHub site repos now have index.html, articles.json, and a slug folder
• Cloudflare Pages auto-deploys within ~30 seconds
• Visit your domain — the site is live with the first article

If something fails:
• Check the Actions log for the exact error message
• Most common issue: wrong repo format in config (must be username/reponame — no URL, no .git)
• Second most common: GitHub token doesn't have repo scope — regenerate with correct permissions
10
Adding New Sites Going Forward
Expanding the network is easy
1. Register the new domain → add to Cloudflare → create a GitHub repo for it
2. Connect the new repo to Cloudflare Pages (same process as Step 4)
3. Come back to this launcher → Step 3 → click the "+" slot to add the site
4. Fill in domain, repo, persona, topics — same as any other site
5. Generate Config again → download the new network-config.json
6. Upload the new JSON to your bot repo (it overwrites the old one)
7. The bot picks up the new site on the next run automatically

No code changes required — the bot reads the config file, so adding a site is entirely config-driven.

What's Next — Scaling & SEO
After your first week of publishing
Submit sitemaps to Google:
Each site auto-generates sitemap.xml. Submit each at search.google.com/search-console → Add property → each domain → Sitemaps → https://yourdomain.com/sitemap.xml

Monitor cross-link growth:
The Dashboard backlinks tab shows a matrix of which sites link to which. After 30 days you should see 20-30+ cross-links per site.

Add Telegram editorial control:
Every Monday at 9am UTC, the bot sends 20 article ideas to your Telegram. Reply with the idea numbers (e.g. "1 4 7 12 16") to queue those articles. Reply "Auto" to let the bot pick the best 7 automatically.

Scale to more sites:
The system handles up to 20 active sites per run within GitHub Actions' free 2,000 minutes/month. Beyond 20, consider upgrading to GitHub Pro or splitting into 2 bot repos.

Newsletter growth:
Add subscription forms to each site using Brevo's free embeddable forms. When readers subscribe, they're added to that site's Brevo list and receive every future article by email automatically.
