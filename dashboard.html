📋 How to Buy a Domain and Connect It - Step by Step

      click to expand
    

    

      

        
PART 1 - Buy a Domain (Cloudflare Registrar - cheapest option)

        
    
          
  1. Go to cloudflare.com and create a free account if you don't have one
    2. 
          
  2. Left sidebar: click Domain Registration → Buy Domains
    3. 
          
  3. Search for your domain name (e.g. onlinebizpro.com)
    4. 
          
  4. Select a domain and complete purchase. Cloudflare charges at cost - no markup
    5. 
          
  5. Domain is automatically added to your Cloudflare account with DNS managed by Cloudflare
    6. 
        

      

      

        
PART 2 - Add Domain to Cloudflare (if bought elsewhere)

        
    
          
  1. Go to cloudflare.com → Add a Site → enter your domain → choose Free plan
    2. 
          
  2. Cloudflare scans existing DNS. Review and click Continue
    3. 
          
  3. Cloudflare gives you 2 nameservers (e.g. alex.ns.cloudflare.com)
    4. 
          
  4. Log in to where you bought the domain (Namecheap, GoDaddy, etc.) and update nameservers to Cloudflare's two
    5. 
          
  5. Wait up to 48h for nameserver propagation. Cloudflare emails you when active
    6. 
        

      

      

        
PART 3 - Connect Domain to Your GitHub Pages Site

        
    
          
  1. Go to your site's GitHub repo → Settings → Pages
    2. 
          
  2. Under Custom domain enter your domain (e.g. onlinebizpro.com) and click Save
    3. 
          
  3. GitHub creates a CNAME file in your repo automatically. Leave it there.
    4. 
          
  4. Now set up DNS in Cloudflare (see Part 4 below), OR use the "Set Up All Custom Domains" button below to automate it
    5. 
          
  5. Wait for the green checkmark in GitHub Pages settings. Then click Enforce HTTPS
    6. 
        

      

      

        
PART 4 - Cloudflare DNS Records (manual option)

        
In Cloudflare DNS panel, add these records for your domain:

        

          CNAME   www   →   siavashsed.github.io   (DNS Only - gray cloud)

          A        @    →   185.199.108.153

          A        @    →   185.199.109.153

          A        @    →   185.199.110.153

          A        @    →   185.199.111.153
        

        
Use DNS Only (gray cloud) - NOT proxied (orange cloud). GitHub Pages handles SSL. If you use orange cloud, set SSL to Flexible in Cloudflare.

      

      

        
PART 5 - Connect to Articles (Auto)

        
Articles automatically use the site's base domain. Once your GitHub Pages custom domain is set:

        - Your homepage: yourdomain.com

        - Each article: yourdomain.com/article-slug/

        - The sitemap: yourdomain.com/sitemap.xml

        No extra config needed - GitHub Pages handles routing automatically.

Guide & Reference

  
Step-by-step startup, every file explained, and key commands


  
  

    
How to Start - Do This Every Time

    

      

        
1

        
Double-click "Start Server.command" on your Desktop inside the content-network folder.
A black terminal window opens and shows "SiaAI Network Server - http://localhost:8765/start" - leave it open while working.

      

      

        
2

        
Browser opens automatically to the Start page at localhost:8765/start.
Green dot = server running. From here click "Open Hub" or "Open Dashboard".

      

      

        
3

        
First time only: add your API keys - click Settings (gear icon, bottom left).
Add Anthropic Key, GitHub Token, Pexels Key. They auto-save and stay forever.

      

      

        
4

        
Set your Bot Repo in Settings - type Siavashsed/kavalsia-bot and click Save.
This connects the hub to your GitHub Actions bot that publishes articles daily.

      

      

        
5

        
Push site templates to GitHub - open Terminal, run:
python3 /path/to/content-network-V2/push-sites.py --push YOUR_TOKEN
This uploads index.html, about.html, legal pages to all 22 GitHub repos. Do once - then bot keeps them updated.

      

      

        
6

        
Trigger first article run - click the green "Run Now" button at the top of this hub.
GitHub Actions will queue then start. Status shows "queued" for ~30 seconds, then "in_progress". Articles appear on the sites within ~5 minutes.

      

      

        
7

        
After that - bot runs automatically every day.
You only need to open the server when you want to use the hub. Articles publish on their own via GitHub Actions even with the server off.

      

    

    

      Important: The server only needs to run when YOU are using the hub. The daily bot runs on GitHub - it doesn't need your computer on.
    

  


  

    

      
Python Files (Backend)

      

        
server.py - run first, always
Starts the Hub at localhost:8765. Proxies Claude, GitHub, Cloudflare APIs. Must be running for Hub to work.
python3 server.py

Config Files

      

        
network-config.json - the brain
All 22 sites: domains, repos, tones, personas, topics, settings. Edit via Hub Settings panel or ↑ Push to sync to GitHub.

        
.env - never commit
API keys for local use. Copy from .env.example and fill in.
cp .env.example .env

Python dependencies. Install once:
pip3 install -r requirements.txt

Deploy to SiteGround - Access from Any Device

      
Deploy this server to SiteGround so hub, dashboard, and all tools are accessible from any computer or phone. Requires GrowBig or GoGeek plan.

      

        

          
1

          
Create subdomain - SiteGround Site Tools → Domain → Subdomains → Create → enter hub → gives you hub.kavalsia.com

        

        

          
2

          
Set up Python App - Site Tools → Devs → Python Manager → Create. Python: 3.11, Startup file: passenger_wsgi.py, Entry point: application

        

        

          
3

          
Upload all files - File Manager or FTP → upload everything from the content-network folder into the subdomain directory

        

        

          
4

          
Install dependencies - Python Manager → your app → Run pip install: flask requests anthropic python-dotenv

        

        

          
5

          
Add environment variable - Python Manager → Environment Variables → add ANTHROPIC_API_KEY

        

        

          
6

          
Restart and visit - click Restart → go to https://hub.kavalsia.com → login wall → done

How to Save

      
Edit any field in the form and click Save. Then click Push Config to send to GitHub. Server must be running to Save to disk.
