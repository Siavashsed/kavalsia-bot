Quick Start - Get the Bot Running
1
Add GitHub Secrets
Go to github.com/Siavashsed/kavalsia-bot → Settings → Secrets → add ANTHROPIC_API_KEY and GH_PAT
2
Trigger a test run
Go to Actions tab → "Daily Content Publisher" → Run workflow → wait ~5 min
3
Check a site
Visit siavashsed.github.io/cryptopulse - if an article appeared, the system works
4
Bot runs daily
Every day at 8am UTC, GitHub Actions publishes one article per site automatically. Zero maintenance.
5
Custom domains (later)
Buy .com domains, add CNAME to GitHub Pages, update domain field in network-config.json
6
Online access (later)
Deploy server.py to Railway.app - then access hub + dashboard from any device, anywhere
All Files - What Each One Does
File What it does Type
hub.html Main UI - AI[S] Terminal, activity feed, site config, backlinks, API keys. Requires server. UI
dashboard.html 22-site control panel - status, visit links, manual post, backlink matrix. Works as file:// too. UI
megadash.html Visual network overview with sparklines, traffic charts, site summaries. UI
editor.html Rich article editor - write manually, push to GitHub repo. UI
start.html This page - home screen with all tools and documentation. UI
server.py Local web server (port 8765). Serves all HTML files, proxies Claude AI + GitHub APIs. Server
bot.py The content publisher. Runs on GitHub Actions daily - generates articles via Claude, pushes to each site repo. Bot
telegram_bot.py Telegram bot - /ask questions, /report on network, /publish articles via chat. Bot
network-config.json Master config - all 22 sites, personas, topics, settings. Edited via Hub or directly. Config
.env Local API keys (ANTHROPIC, PEXELS, GITHUB_TOKEN). Never committed to GitHub. Config
Start Server.command Double-click on Mac to start server.py and open browser automatically. Server
.github/workflows/ daily-post.yml - runs bot.py at 8am UTC. weekly-ideas.yml - sends Telegram ideas Monday 9am UTC. Bot
